# 小红书运营策略总结

## 背景
方向：生活育儿 + AI提效探索，利用碎片时间创作。当前问题：内容太浅、创作耗时、封面无模板、不了解推流逻辑。

---

## 一、内容质量：从"分享一个点"变成"可抄作业的教程"

- 小红书不需要"深入"，需要**具体+有用+可复制**
- 每篇三要素缺一不可：**具体场景 + 完整操作路径 + 结果对比**
- 标题公式示例："1个prompt帮我30秒验完商家话术"
- 反例：写"AI可以验证商家说辞"是信息；写"我花30秒用AI拆穿了某鱼商家的话术，附完整prompt"是内容

### 选题筛选标准（每项0-2分，≥4分才值得发）
1. **痛点普遍性** — 很多人都会遇到这个场景吗？
2. **效果可感知** — 有没有明确的 before/after？
3. **可复制性** — 读者看完能立刻照做吗？

### 内容模板（写进 Obsidian agent prompt）
```
- 标题/Hook：[一句话，制造好奇或冲突]
- 痛点场景：[读者什么时候会遇到这个问题]
- AI 解法：[分步骤，每步配截图说明，不超过5步]
- 效果对比：[before/after 或 时间对比]
- CTA：[用提问句引导评论，不用"关注我"]
```

### 写作规则（写进 agent prompt）
- 标题必须包含具体数字或时间（如"30秒"、"1个prompt"）
- 第一句必须是痛点场景，不是自我介绍
- 每个步骤配截图说明，不超过5步
- 结尾用提问句引导评论
- 语气口语化，像跟朋友说话，禁止用"赋能""干货满满"

### 关于学习好笔记
- 不要把范文扔给 agent "学习"，agent 只会模仿表面语气词
- 正确做法：自己找5-10篇好笔记，逐篇标注"好在哪"，归纳成规则写进 prompt

---

## 二、创作流程优化

### 瓶颈根因
点子写太详细 → agent 转化质量不够 → 反复改

### 优化方案
1. **点子阶段只记三行**：场景、操作、结果（不要写详细）
2. **Obsidian 中建发布模板**（上面的内容模板），让 agent 按模板填充
3. **审核阶段只改两件事**：hook 那句话 + 有没有说人话，其他不改
4. 完美主义是碎片时间创作的最大敌人

---

## 三、封面图

### 核心原则
把"每次设计"变成"一次性建模板，之后只填充"

### 做法
1. 先研究5-10个同赛道博主的封面，提炼结构规则（不是模仿设计）
2. 分析三个维度：信息结构、文字公式、视觉风格
3. 从中选**不超过2套模板**做成 Canva 模板：
   - 1套教程类（纯色底+大字标题）
   - 1套效果展示类（截图+对比）
4. 每次只换文字和截图，2分钟出一张
5. 标题公式：**数字+动作+结果**

### 笔记内图
- 截图 + 系统自带标注（红框/箭头），不做任何美化
- 多步骤按顺序编号1234，不需要拼图排版
- 全程不超过5分钟
- 原始截图比精美排版更有信任感

### 视觉锤方案
- **不做数字分身**，制作成本高且与"压缩封面时间"目标矛盾，AI虚拟形象也不再有差异化
- **用"固定配色+字体+排版"本身作为视觉锤**，成本最低，刷多了用户自然形成记忆
- 可选增强：设计一个简单的个人logo/icon角标，固定在封面某个角落，一次设计永久复用
- 等账号做到一定体量再考虑人格化升级（出镜或数字分身）

## 四、小红书推流逻辑

### 核心机制
- **CES 评分** = 点赞×1 + 收藏×1 + 评论×4 + 转发×4 + 关注×8
- 评论和转发权重远高于点赞 → 内容要设计成"让人想说话"
- **前2小时是关键期**，互动率低于3%会被停止推荐
- **超过65%的流量来源于搜索** → 每篇必须布局搜索关键词

### 具体行动
- 标题和正文布局关键词（如"AI育儿"、"ChatGPT验货"、"AI提效工具"）
- 发布后主动在评论区抛问题
- 每篇结尾用具体问题引导评论（如"你最想用AI解决育儿中的什么问题？"）
- 发布时间固定在你能互动的时段，发布后保证30分钟回复评论

### 发帖频率
- 新号阶段：**每周3-4篇**，质量优先于数量
- 固定发布日（如周二、四、六）
- 每周配比：2篇AI提效实操 + 1篇育儿×AI交叉内容
- 等单篇生产时间压到30分钟以内后再提频
- **绝不为凑频率发低质量内容**，一篇低质笔记拉低的是账号权重
